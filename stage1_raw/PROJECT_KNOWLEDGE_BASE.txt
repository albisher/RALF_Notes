### SUMMARY
A comprehensive knowledge base for the **HMRS Window Cleaning Drone Swarm**, detailing architecture, API, and workflow for autonomous multi-drone coordination in window cleaning simulations.

### DETAILS
This project provides a **heterogeneous drone swarm simulation platform** for window cleaning, enabling vendor-agnostic drone integration, real-time 3D visualization, and ML-driven swarm optimization. It simulates physics, weather, and sensor data using PyBullet and integrates with a Flask backend for RESTful API control. The system supports session persistence via PostgreSQL and Docker orchestration for containerized deployment.

### KEY_FUNCTIONS
- **Swarm Controller**: Orchestrates multiple drones (DJI, Parrot, custom) for coordinated cleaning tasks.
- **Multi-Use Platform**: Dynamically integrates drones from different vendors via standardized APIs.
- **Window Cleaning Focus**: Specialized for facade maintenance with LiDAR and computer vision mapping.
- **Session Persistence**: Stores and replays drone operations for analysis.
- **Real-Time Visualization**: Uses Plotly.js and Leaflet for interactive 3D/2D drone swarm tracking.

### DEPENDENCIES
- **Backend**: Flask, Socket.IO, PyBullet, PostgreSQL, NumPy, SciPy
- **Frontend**: Vue.js 3, Plotly.js, Leaflet, Socket.IO Client
- **Infrastructure**: Docker Compose, Nginx, PostgreSQL 15

### USAGE
1. Deploy via Docker Compose (`docker compose up`).
2. Access the Flask API at `http://localhost:5007/api/`.
3. Use Socket.IO for real-time drone control and visualization.
4. Integrate custom drones via vendor-specific adapters in the swarm controller.
