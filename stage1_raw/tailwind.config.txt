### TYPE

### SUMMARY
Configures Tailwind CSS for a Vue/JS application with custom dark/light themes, RTL support, and extended color palette.

### DETAILS
This `tailwind.config.js` file defines Tailwind CSS settings for a project, including:
- **Content paths** for HTML and source files (`.vue`, `.js`, `.ts`, etc.) to scan for classes.
- **Dark mode** configured via the `darkMode: 'class'` option, allowing dynamic toggling via CSS classes.
- **Extended theme** with custom colors for UI elements (backgrounds, borders, buttons, text) for both light and dark themes, sourced from a `colors.txt` file.
- **Plugins** like `tailwindcss-rtl` for right-to-left language support.
- **Core plugins** to ensure Tailwind’s default styles (`preflight`) are applied.

The config maps Tailwind’s default palette with custom values for a cohesive UI system.

### KEY_FUNCTIONS
- **`content`**: Specifies files to process for Tailwind class detection.
- **`darkMode`**: Enables dynamic dark mode via CSS classes (e.g., `dark:` prefix).
- **`theme.extend.colors`**: Defines custom color variables for UI components (e.g., `main-bg`, `btn-primary`).
- **`plugins`**: Adds RTL support via `tailwindcss-rtl`.
- **`corePlugins.preflight`**: Ensures Tailwind’s base styles are applied globally.

### DEPENDENCIES
`tailwindcss`, `tailwindcss-rtl`

### USAGE
1. Install dependencies:
   ```bash
   npm install tailwindcss postcss autoprefixer tailwindcss-rtl
   ```
2. Run Tailwind’s build script:
   ```bash
   npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch
   ```
3. Use the extended colors in `.vue`/`.js` files via Tailwind classes (e.g., `bg-main-bg` or `text-text-primary`).

### CALLOUTS
>[!INFO]- Dark Mode Integration
> Tailwind’s `darkMode: 'class'` relies on a `dark:` prefix in CSS (e.g., `.dark { @apply bg-dark-main-bg; }`). Ensure your app dynamically toggles this class (e.g., via a theme switcher).

>[!WARNING]- Color Conflicts
> Overlapping color names (e.g., `text-secondary` vs. `border-dark-medium`) may cause conflicts. Use unique aliases (e.g., `text-secondary` vs. `border-input`) to avoid ambiguity.
