### CALLOUTS
>[!INFO]- **Single Responsibility Principle**
> Each box (e.g., `MLControllerBox`) fulfills **one purpose** (e.g., neural network control) and avoids mixing concerns. Example: `PathPlannerBox` handles only path planning, not sensor data.

>[!WARNING]- **Internal Weight Exposure**
> Minor compliance risk: `MLControllerBox` exposes weights (`self.W1`, `self.W2`) via direct state access. Encapsulate these for stricter OOP adherence.
