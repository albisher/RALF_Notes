### SUMMARY
Summary of all external integrations for HMRS simulation system, detailing OSM, Google Maps, weather, and drone SDK implementations.

### DETAILS
This document outlines the complete implementation of external integrations for the HMRS simulation system. It covers four key areas: **3D building data** (via OpenStreetMap and Google Maps), **real-time weather** (via OpenWeatherMap API), and **drone control** (via DJI SDK). Each integration includes data fetching, parsing, and conversion logic tailored to the simulation environment, with rate limiting, caching, and fallback mechanisms for robustness. The implementations are modular, located in dedicated Python modules within the `simulation/swarm/integrations` directory.

### KEY_FUNCTIONS
- **OSMBuildingsIntegration.get_buildings_in_radius()**: Fetches and processes OSM building data within a specified radius.
- **GoogleMaps3DIntegration.get_tileset_url()**: Retrieves 3D tileset URLs for photorealistic rendering.
- **WeatherAPIIntegration.get_current_weather()**: Fetches real-time weather data and converts it to simulation-compatible formats.
- **create_adapter()**: Creates a unified adapter for DJI SDK integration, handling connection and command translation.

### DEPENDENCIES
- `overpass-api` (for OSM data fetching),
- `requests` (for HTTP API calls),
- `CesiumJS` (for 3D tiles rendering),
- `DJI SDK` (for drone control, optional runtime dependency).

### USAGE
1. Import the integration module (e.g., `from swarm.integrations import OSMBuildingsIntegration`).
2. Initialize the integration class and call methods like `get_buildings_in_radius()` or `get_tileset_url()` with required coordinates.
3. For weather, check availability (`is_available()`) before calling `get_current_weather()`.
4. For drones, ensure the DJI SDK is installed; the adapter gracefully degrades if missing.

### CALLOUTS
>[!INFO]- Important Note
> All OSM and weather integrations include rate-limiting to avoid API abuse. The DJI SDK adapter requires manual SDK installation; the system defaults to a mock adapter if the SDK is unavailable.

>[!WARNING]- Caution
> Google Maps 3D Tiles API requires a valid API key. Without it, the integration will return `None` for tileset URLs. Weather API calls may fail if the OpenWeatherMap API key is invalid or expired.
