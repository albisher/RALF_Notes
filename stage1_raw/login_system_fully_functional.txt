### SUMMARY
A fully functional login system with real API integration, conditional UI rendering, and test user credentials for development.

### DETAILS
This implementation resolves a login system with three core requirements: a full-screen login page, test user credentials (`test/passtest`), and conditional sidebar visibility post-login. The system uses Vue.js with Pinia for state management, integrates a real backend API for authentication via JWT tokens, and dynamically adjusts UI elements based on authentication status. The backend includes a test user setup script, while the frontend modifies `App.vue` to conditionally render navigation components and ensures seamless API communication.

### KEY_FUNCTIONS
- **`App.vue`**: Conditional rendering of UI elements (sidebar/app bar) based on authentication state and route.
- **`Auth Store`**: Manages JWT token handling, user state, and API authentication logic with error/loading states.
- **`create_test_user()`**: Backend function to ensure test credentials (`test/passtest`) exist for testing.
- **`login()`**: Auth store method calling `/api/auth/login` to generate tokens and update user state.
- **`Login.vue`**: Displays the full-screen login form with embedded test credentials hint.

### USAGE
1. **Access Login Page**: Navigate to `/login` (e.g., `https://localhost:8443/login`).
2. **Use Test Credentials**: Enter `test` as username and `passtest` as password for quick testing.
3. **Post-Login**: Redirects to dashboard with a sidebar (hidden during login).
4. **API Endpoints**: Use `/api/auth/login`, `/api/worlds`, `/api/dashboard`, etc., with JWT headers for authenticated requests.
